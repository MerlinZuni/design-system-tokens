### ColorSwatch.mdx

Purpose: "ColorSwatch displays a color token visually as a square swatch with a label."
When to use: "Use ColorSwatch in documentation pages to preview color tokens. It is a documentation primitive, not intended for production UI."

Canvas embeds: Default, AllSizes, Grid

---

### Surface.mdx

Purpose: "Surface renders a semantic container with background, border, and elevation from the design token system."
When to use: "Use Surface when you need a container whose visual properties adapt to the active brand and color mode. Pair `elevation` with `background` to create distinct visual layers."

Canvas embeds: Default, WithElevation, WithBorder, Inverse
Note on inverse: "When using `background='inverse'`, ensure child text uses `var(--dsx-color-text-inverse)` for contrast."

---

### Stack.mdx

Purpose: "Stack arranges children vertically with consistent spacing from the design token scale."
When to use: "Use Stack for any vertical layout — form fields, card content, page sections. The `gap` prop maps to spacing tokens, ensuring consistent rhythm."

Canvas embeds: Default, AllGaps, WithAlignment

---

### Inline.mdx

Purpose: "Inline arranges children horizontally with wrapping and consistent spacing."
When to use: "Use Inline for horizontal layouts like tag lists, button groups, or navigation items. Children wrap automatically when they exceed the container width."

Canvas embeds: Default, Wrapping, Justified

---

### VisuallyHidden.mdx

Purpose: "VisuallyHidden renders content invisible to sighted users but accessible to screen readers."
When to use: "Use VisuallyHidden to provide screen reader labels for icon-only buttons, decorative images with context, or skip navigation links. Never use it to hide content that should be visible."

Canvas embeds: Default, Focusable
Accessibility note: "VisuallyHidden uses the standard CSS clip pattern recommended by WCAG. Content is positioned absolutely at 1x1px with overflow hidden."
