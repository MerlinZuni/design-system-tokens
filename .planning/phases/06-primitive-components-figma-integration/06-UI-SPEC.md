# Phase 6 — UI Design Contract

> Visual and interaction contract for Phase 6: Primitive Components & Figma Integration.
> Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Context Note

This phase produces **library components** consumed by Storybook, not a product UI. The "UI" surfaces are:
1. The component render output inside Storybook story canvases.
2. The MDX documentation pages in the Storybook sidebar under `Primitives/`.

All visual tokens are consumed via `--dsx-*` CSS custom properties set on the `data-brand`/`data-theme` root by the existing `withThemeAttributes` global decorator. No new design tokens are introduced. The contract below governs how components consume existing tokens and how stories present them.

## Spacing Scale

Declared values sourced from `--dsx-spacing-*` in `packages/tokens/dist/css/tokens.css`.

Components use CSS var references, not raw pixel values.

| Token | CSS Var | Raw Value | Usage in Phase 6 |
|-------|---------|-----------|------------------|
| xs | `--dsx-spacing-1` | 4px | Icon gaps inside ColorSwatch label, inline padding |
| sm | `--dsx-spacing-2` | 8px | Compact child spacing in Stack/Inline |
| md | `--dsx-spacing-4` | 16px | Default gap for Stack/Inline; Surface internal padding |
| lg | `--dsx-spacing-6` | 24px | Section padding in MDX story canvases |
| xl | `--dsx-spacing-8` | 32px | Layout gaps between story sections in MDX |
| 2xl | `--dsx-spacing-12` | 48px | Major section breaks in MDX docs |
| 3xl | `--dsx-spacing-16` | 64px | Page-level MDX top/bottom padding |

Exceptions:
- Touch target minimum: 44px (`--dsx-spacing-11`) for any interactive story control wrapper.
- ColorSwatch size variants: `sm` = 32px, `md` = 48px, `lg` = 64px — these are component-internal sizes, sourced from spacing scale.

All gap/padding values in component inline styles must reference CSS vars, not hardcoded px values.

## Typography

Sourced from `--dsx-typography-*` in `packages/tokens/dist/css/tokens.css`. Font is Archivo throughout.

| Role | CSS Step | Size | Weight | Line Height | Usage |
|------|----------|------|--------|-------------|-------|
| Caption / Label | `text-xs` | 12px | 400 | 16px (1.33) | ColorSwatch token label, story annotation text |
| Body | `text-base` | 16px | 400 | 24px (1.5) | MDX prose, component usage descriptions |
| Heading-sm | `text-xl` | 20px | 500 | 28px (1.4) | MDX section headings (h3, h4 in docs) |
| Heading-lg | `text-2xl` | 24px | 500 | 32px (1.33) | MDX page titles (h1, h2) |

**Weights used: exactly 2 — regular (400) and medium (500).** No other weights.

**Text component variant-to-token mapping** (Claude's Discretion area):

| Variant prop | CSS Step | Rendered element default |
|-------------|----------|--------------------------|
| `display` | `text-9xl` | `span` |
| `heading-xl` | `text-4xl` | `span` |
| `heading-lg` | `text-3xl` | `span` |
| `heading-md` | `text-2xl` | `span` |
| `heading-sm` | `text-xl` | `span` |
| `body-lg` | `text-lg` | `span` |
| `body-md` | `text-base` | `span` (default) |
| `body-sm` | `text-sm` | `span` |
| `label` | `text-sm` | `span` |
| `caption` | `text-xs` | `span` |
| `code` | `text-sm` | `span` |

All variants consume `--dsx-typography-{step}-{property}` CSS vars via inline styles.

## Color

All color values are consumed via semantic `--dsx-*` CSS custom properties, not hardcoded hex values. The 60/30/10 split below describes token role, not specific hex codes, because the active palette changes with brand/mode.

| Role | CSS Var | Light mode resolved value | Usage |
|------|---------|--------------------------|-------|
| Dominant surface (60%) | `--dsx-color-background-default` | `#ffffff` (neutral-0) | Storybook story canvas background, component default background |
| Secondary surface (30%) | `--dsx-color-background-subtle` | `#f9fafb` (neutral-50) | MDX docs background panels, story wrapper sections |
| Accent (10%) | `--dsx-color-action-default` | `#4FC4C4` (brand-500/teal) | Focus ring on VisuallyHidden focus-visible; ColorSwatch border on hover |
| Destructive | `--dsx-color-status-error-*` | red-500 family | Not applicable — Phase 6 has no destructive actions |
| Text default | `--dsx-color-text-default` | `#111827` (neutral-900) | All component text |
| Text muted | `--dsx-color-text-muted` | neutral-600 | Captions, token labels |
| Border default | `--dsx-color-border-default` | neutral-200 | Surface component border, ColorSwatch border |
| Focus ring | `--dsx-color-focus-ring` | brand-500 (#4FC4C4) | Keyboard focus outline on all focusable primitives |

**Accent reserved for:** keyboard focus ring (`--dsx-color-focus-ring`), ColorSwatch hover border, and story "live value" highlight spans in MDX. Not used for general interactive elements beyond these three specific cases.

**Theme reactivity:** All color values update automatically when `data-brand`/`data-theme` attributes change via the global `withThemeAttributes` decorator. Components must not hardcode hex values.

## Component Visual Contracts

### Text

- Renders inline styles only using `--dsx-typography-{step}-*` CSS vars.
- No background, border, or padding by default.
- `color` not set by component — inherits from parent or caller sets it explicitly.
- `as` prop defaults to `span`.

### ColorSwatch

- Square swatch box + token name label beneath.
- Swatch background: inline style `background-color: var(--dsx-color-{tokenPath})` passed via `tokenVar` prop.
- Border: `1px solid var(--dsx-color-border-default)`.
- Border on hover: `1px solid var(--dsx-color-action-default)` (accent).
- Label: 12px Archivo 400, `--dsx-color-text-muted`.
- Size variants: `sm` = 32×32px, `md` = 48×48px (default), `lg` = 64×64px.
- No `as` prop — always renders `div` + `span`.

### Surface

- Renders a block container with semantic background, optional border, optional shadow.
- Props: `elevation` (`none` | `sm` | `md` | `lg`), `background` (`default` | `subtle` | `card` | `inverse`), `border` (boolean), `as` (default `div`).
- `background` maps to `--dsx-color-background-*` and `--dsx-color-surface-*` vars.
- `elevation` maps to `--dsx-shadow-*` vars (composite shadow constructed from offset-x/y + blur + spread + color sub-vars).
- `border` renders `1px solid var(--dsx-color-border-default)`.
- Border radius: `4px` (no spacing token for border-radius — use raw 4px).

### Stack

- Vertical flex container (`flex-direction: column`).
- `gap` prop accepts spacing token key: `xs` | `sm` | `md` | `lg` | `xl` | `2xl` | `3xl` — maps to `--dsx-spacing-*` values above.
- `align` prop: `start` | `center` | `end` | `stretch` (default: `stretch`).
- `as` defaults to `div`.
- No background, border, padding by default.

### Inline

- Horizontal flex container (`flex-direction: row`, `flex-wrap: wrap`).
- Same `gap` and `align` props as Stack. Additional `justify` prop: `start` | `center` | `end` | `between` (default: `start`).
- `as` defaults to `div`.
- No background, border, padding by default.

### VisuallyHidden

- Standard CSS clip pattern (position absolute, 1×1px, overflow hidden, clip rect(0,0,0,0)).
- Renders `span` always — no `as` prop.
- No visual output. Focus-visible state on children: `outline: 2px solid var(--dsx-color-focus-ring)`.

## Copywriting Contract

Phase 6 produces no end-user product UI. All copy is developer-facing documentation in Storybook MDX files.

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — no interactive product UI in this phase |
| Empty state — no children | "Add children to see content" (story canvas placeholder for Stack/Inline/Surface) |
| Empty state — no token | "Pass a tokenVar prop to display a color swatch" (ColorSwatch story placeholder) |
| Error state — build error | "Component failed to render — check browser console for details" (story error boundary message) |
| Destructive confirmation | Not applicable — Phase 6 has no destructive actions |

**MDX documentation copywriting conventions (Claude's Discretion):**

- **When to use section:** 2–3 sentences. Lead with the use case. Example: "Use `Text` when you need to apply a design system type scale step to any text content."
- **Variant description:** Name the variant, state the semantic HTML recommendation. Example: "`heading-lg` — Use with `as='h2'` for second-level section headings."
- **Prop description (JSDoc):** One sentence. Verb phrase. Example: `/** Typography variant — selects a type scale step. */`
- **Code Connect placeholder comment:** `// TODO: replace with Figma component node ID from Dev Mode`

## Story Coverage Contract

Each component must have the following named story exports:

| Component | Required Named Stories |
|-----------|----------------------|
| Text | `Default` (body-md), `AllVariants` (render stack of all 11 variants), `WithSemanticElement` (as='h2' heading-lg), `Truncated` (overflow text) |
| ColorSwatch | `Default` (md size, brand-500), `AllSizes` (sm/md/lg), `SemanticToken` (background-default), `Grid` (multiple swatches in Inline) |
| Surface | `Default` (no elevation, default bg), `WithElevation` (all 3 levels), `Inverse` (background='inverse'), `WithBorder` |
| Stack | `Default` (md gap, 3 children), `AllGaps` (render each gap value labeled), `WithAlignment` (align variants) |
| Inline | `Default` (sm gap, 4 children), `Wrapping` (many children to show wrap), `Justified` (justify='between') |
| VisuallyHidden | `Default` (hidden text next to visible label), `Focusable` (button with visually hidden label) |

All story files include `tags: ['autodocs']` in meta (D-05).

## MDX Documentation Structure

Each MDX file follows this structure (D-03):

```
1. <Meta of={ComponentStories} />
2. # ComponentName
3. One-sentence purpose description.
4. ## When to use
   2–3 sentence guidance.
5. ## Primary example
   <Canvas of={ComponentStories.Default} />
6. ## Variants / States
   <Canvas of={ComponentStories.AllVariants} /> (or relevant variant story)
7. ## Props
   <ArgTypes of={ComponentStories} />
8. ## Figma
   <Figma url="https://figma.com/..." collapsable />
```

The `<Figma>` import path is `storybook-addon-designs/blocks` (confirmed pattern from Phase 5, STORY-11).

## Accessibility Contract

- All focusable elements must have `outline: 2px solid var(--dsx-color-focus-ring)` on `:focus-visible`.
- `VisuallyHidden` is the mechanism for providing screen reader labels — its children must be meaningful text.
- `Text` component with `as` prop: caller is responsible for correct heading hierarchy — component does not enforce it.
- `ColorSwatch` must include `aria-label` containing the token name and resolved value (passed as prop).
- `Surface` with `background='inverse'` must pair with `--dsx-color-text-inverse` for text inside it — this is a documentation guideline in the MDX, not enforced by the component.
- `VisuallyHidden` stories must include a `FocusVisible` story showing the focus ring pattern.
