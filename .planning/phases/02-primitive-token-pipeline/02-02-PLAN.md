<!-- Naming conventions locked in CONTEXT.md decisions. -->

Token naming (per D-03, D-05, D-12, D-14, D-18, D-21):
- Color:      `color.{hue}.{step}`          → CSS: `--dsx-color-{hue}-{step}`
- Spacing:    `spacing.{step}`               → CSS: `--dsx-spacing-{step}`
- Typography: `typography.{step}`            → CSS: `--dsx-typography-{step}-font-size`, etc.
- Font family:`font.family.{variant}`        → CSS: `--dsx-font-family-{variant}`
- Elevation:  `elevation.{level}`            → CSS: `--dsx-elevation-{level}-...`
- Grid:       `grid.breakpoint.{name}`       → TS:  `GridBreakpointSm`, `GridBreakpointMd`, etc.

Standard interface hues required (per D-02):
  red, green, yellow, blue, orange, purple — steps 50, 100, 200, 300, 400, 500, 600, 700, 800, 900, 950

Tailwind spacing steps required (per D-05):
  0, 0.5, 1, 1.5, 2, 2.5, 3, 3.5, 4, 5, 6, 7, 8, 9, 10, 11, 12, 14, 16, 20, 24, 28, 32, 36, 40, 44, 48, 52, 56, 60, 64, 72, 80, 96

Tailwind type scale steps required (per D-12):
  text-xs, text-sm, text-base, text-lg, text-xl, text-2xl, text-3xl, text-4xl, text-5xl, text-6xl, text-7xl, text-8xl, text-9xl

Breakpoints required (per D-16):
  sm, md, lg, xl, 2xl

Font families required (per D-07, D-08, D-09):
  font.family.default = ["Archivo", "sans-serif"]
  font.family.variable = ["Inter Variable", "Inter", "sans-serif"]

DTCG group-level $type inheritance (from research Pattern 3):
  Set $type once at the group level — child tokens inherit it, no need to repeat on every leaf.

Composite token authoring format (from research Pattern 3):
  - Typography: $type: "typography", $value: { fontFamily, fontWeight, fontSize, lineHeight, letterSpacing }
  - Shadow:     $type: "shadow", $value: { color, offsetX, offsetY, blur, spread } (or array of these)
  Note: offsetX/offsetY (not x/y) — use DTCG standard format, not Tokens Studio legacy format
