## Spacing Scale

8-point scale applied to documentation UI layout. The token system's own spacing values (`--dsx-space-component-*`, `--dsx-spacing-*`) are used only inside the token visualization specimens — not for documentation page layout.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label-to-value padding in TokenTable rows |
| sm | 8px | Intra-row padding in TokenTable, color swatch caption gap |
| md | 16px | Default element spacing, card inner padding, section header gap |
| lg | 24px | Section padding (primitives vs semantic groupings within a page) |
| xl | 32px | Between major sections on a token page |
| 2xl | 48px | Between category headings and their content blocks |
| 3xl | 64px | Top-of-page padding before first section |

Exceptions:
- SpacingBar component: bar height is exactly the spacing value it represents (e.g. the `spacing-4` bar is 16px tall). This is a specimen exception — not layout spacing.
- ElevationCard: minimum height 96px to accommodate shadow visibility beneath the card.
- Breakpoint ruler: minimum total width 640px to represent the full breakpoint range at readable scale.

## Typography

Documentation UI typography (Storybook page chrome, MDX prose, token labels).

| Role | Size | Weight | Line Height | Usage |

## Color

Documentation page chrome uses the design system's own semantic tokens so the documentation demonstrates the live token system. Light mode by default (D-04), reactive to the global theme switcher (D-05, D-09).

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--dsx-color-background-default)` — resolves to `#ffffff` in parent/light | Page canvas background, MDX prose background |
| Secondary (30%) | `var(--dsx-color-surface-panel)` — resolves to `#f8f8f8` in parent/light | TokenTable row alternates, alias chain block background, specimen card backgrounds |
| Accent (10%) | `var(--dsx-color-action-default)` — resolves to `#4fc4c4` (teal) in parent/light | Active toolbar dropdown state indicator; alias chain arrow glyphs; "copied" feedback flash on click-to-copy |
| Destructive | `var(--dsx-color-status-error-text)` — resolves to `#b91c1c` in parent/light | Token resolution error state text (when a CSS var fails to resolve to a non-empty value) |

Accent reserved for:
1. Toolbar dropdown active state border/indicator
2. Alias chain resolution arrows (→) in the SemanticToken alias chain display
3. Click-to-copy confirmation flash (300ms background flash on token name or CSS variable copy)

No other elements use accent color. Interactive hover states use `var(--dsx-color-action-hover)` — not raw accent hex.

All color values reference semantic CSS variables. The documentation page visually demonstrates the theme switcher working on itself.

## Interaction Contracts

### Theme Switcher (D-01 through D-05)

Two independent toolbar dropdowns rendered by Storybook's `globalTypes` API in `preview.tsx`.

| Control | Global Key | Values | Default |
|---------|-----------|--------|---------|
| Brand dropdown | `brand` | `parent` "Parent Brand", `child` "Child Brand" | `parent` |
| Mode dropdown | `mode` | `light` "Light", `dark` "Dark" | `light` |

Decorator applies both as data attributes on `document.documentElement`:
- `data-brand="{brand}"` and `data-theme="{mode}"` must always be set together.
- If either attribute is missing, NO semantic CSS variables resolve (compound selector requirement from D-02).
- Changing either dropdown immediately re-renders all pages without navigation (persistent global per D-05).

Implementation: `useEffect` inside `withThemeAttributes` decorator reacts to both globals on every change.

### Token Page — Click-to-Copy

Every token name cell and every CSS variable cell in TokenTable rows is click-to-copy.

| Target | On click | Feedback |
|--------|----------|----------|
| Token name (e.g. `color.background.default`) | Copy dotted token name to clipboard | 300ms `var(--dsx-color-highlight-selected)` background flash, then reset |
| CSS variable (e.g. `--dsx-color-background-default`) | Copy CSS variable string to clipboard | Same 300ms flash |
| Hex/raw value | Copy raw value to clipboard | Same 300ms flash |

No tooltip needed — flash is sufficient affordance. No destructive actions in this phase.

### Token Page — Section Structure (D-08)

Each token category page (Colors, Typography, Spacing, Elevation, Grid) follows this structure:

1. Display heading (28px, weight 600)
2. Brief description paragraph (14px, weight 400, `var(--dsx-color-text-muted)`)
3. "Primitive Tokens" section heading (20px, weight 600) with count badge (label 12px)
4. Primitive token table/visualization
5. "Semantic Tokens" section heading (20px, weight 600) — only visible when semantic tokens exist for this category
6. Semantic token table with alias chain column — live-reacts to theme switcher (D-09)

### Styles Sub-Pages (D-10 through D-13)

Five sub-pages under `Styles/` in the sidebar, ordered as follows:

1. `Styles/Headings` — heading hierarchy h1–h6
2. `Styles/Body Text` — body, caption, overline, label variants
3. `Styles/Surfaces` — surface/card patterns (background + border + elevation)
4. `Styles/Interactive States` — default, hover, active, focus, disabled
5. `Styles/Feedback` — success, warning, error, info patterns

Each sub-page shows: visual specimen + TokenTable listing every token used in that specimen (CSS variable name + resolved value). Responds to global theme switcher.

## Component Inventory

Custom React components required. All authored in `apps/storybook/stories/components/`.

| Component | Purpose | Key Props |
|-----------|---------|-----------|
| `TokenTable` | Tabular display of token rows with alias chain column | `tokens: SemanticToken[]`, `showAliasChain?: boolean` |
| `ColorSwatch` | Single color tile with hex label | `cssVar: string`, `tokenName: string`, `size?: "sm" \| "md"` |
| `TypographySpecimen` | Renders a paragraph/heading at the token's declared size/weight/line-height | `cssVars: TypographyTokenVars` |
| `SpacingBar` | Horizontal bar whose height equals the spacing value it represents | `cssVar: string`, `tokenName: string`, `pxValue: number` |
| `ElevationCard` | Card with shadow applied via reassembled box-shadow from decomposed shadow sub-vars | `level: "sm" \| "md" \| "lg"` |
| `BreakpointRuler` | Visual proportional ruler of breakpoint ranges using TS constants from `@design-system-x/tokens` | `breakpoints: Record<string, number>` |
| `AliasChain` | Inline display: `semantic.token → primitive.token → #rawvalue` | `chain: string[]` |

`@storybook/blocks` `ColorPalette` and `ColorItem` used for primitive color swatches (adequate for static display). Custom `ColorSwatch` + `TokenTable` used for semantic color rows with alias chain.
