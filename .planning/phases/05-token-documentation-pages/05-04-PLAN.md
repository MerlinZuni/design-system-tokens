**1. ElevationCard.tsx** (`apps/storybook/stories/components/ElevationCard.tsx`)

Renders a card element with the shadow applied visually, plus a detail panel listing all decomposed shadow sub-properties.

```typescript
import React from 'react'

import type { ElevationTokenRow, ShadowLayer } from './token-data'

interface ElevationCardProps {
  token: ElevationTokenRow
}
```

Layout: card container with two sections.

**Visual section:**
- A `div` with:
  - `width: 200px`, `minHeight: 96px` (per UI spec minimum)
  - `backgroundColor: var(--dsx-color-background-surface)`
  - `borderRadius: 8px`
  - `boxShadow: token.boxShadowString` — uses the pre-assembled string from token-data.ts
  - Centered text showing the level name (e.g. "sm", "md", "lg") in `fontSize: 20px`, `fontWeight: 600`
  - `margin: 24px` — extra margin around card so shadow is visible against page background

**Detail section** (below or beside the card):
- Token name (e.g. "elevation.sm") in monospace, wrapped in CopyToClipboard
- CSS var prefix (e.g. "--dsx-shadow-sm-*") in monospace, muted
- Description text
- For each shadow layer, show a table row:
  - Layer label (e.g. "Layer 1", or just the single layer for sm)
  - Each sub-property: color, offset-x, offset-y, blur, spread — with CSS var name and raw value
- Full box-shadow string: the assembled `boxShadowString` in a code block, wrapped in CopyToClipboard

Shadow sub-property CSS var names follow this pattern:
- Single-layer (sm): `--dsx-shadow-sm-color`, `--dsx-shadow-sm-offset-x`, `--dsx-shadow-sm-offset-y`, `--dsx-shadow-sm-blur`, `--dsx-shadow-sm-spread`
- Multi-layer (md): `--dsx-shadow-md-1-color`, `--dsx-shadow-md-1-offset-x`, etc. and `--dsx-shadow-md-2-*`

Container styling: `display: flex`, `flexDirection: column`, `gap: 16px`, `padding: 24px`, `borderBottom: 1px solid var(--dsx-color-border-subtle)`.

**1. BreakpointRuler.tsx** (`apps/storybook/stories/components/BreakpointRuler.tsx`)

A visual proportional ruler showing breakpoint ranges. The ruler total width represents the largest breakpoint, and each segment shows its proportional size.

```typescript
import React from 'react'
import { CopyToClipboard } from './CopyToClipboard'

interface BreakpointRulerProps {
  breakpoints: Record<string, string> // { sm: '640px', md: '768px', ... }
}
```

Implementation:
- Parse px values to numbers: `parseInt('640px')` -> 640
- Sort breakpoints by value: sm (640), md (768), lg (1024), xl (1280), 2xl (1536)
- Total ruler width: use the largest value (1536) as 100%
- For each breakpoint, render a segment:
  - The segment starts at the previous breakpoint value (or 0 for the first) and ends at the current value
  - Width = `(current - previous) / maxValue * 100%`
  - Display: colored bar with breakpoint name label above, px value below
  - Alternating colors: use brand and secondary to differentiate adjacent segments
    - Odd segments: `var(--dsx-color-action-default)` (teal)
    - Even segments: `var(--dsx-color-secondary-300)` (purple-light)
- Minimum total width: 640px (per UI spec)
- Below the ruler, show a reference table:

| Breakpoint | TS Constant | Value |
|---|---|---|
| sm | `GridBreakpointSm` | 640px |
| md | `GridBreakpointMd` | 768px |
| lg | `GridBreakpointLg` | 1024px |
| xl | `GridBreakpointXl` | 1280px |
| 2xl | `GridBreakpoint2xl` | 1536px |

The table maps breakpoint names to their TypeScript constant export names from `@design-system-x/tokens`.

Labels above each segment: breakpoint name in `fontSize: 14px`, `fontWeight: 600`. Labels below: px value in `fontSize: 12px`, `fontFamily: monospace`, `color: var(--dsx-color-text-muted)`.

Segment bars: `height: 40px`, `display: inline-block`, labels centered within segment.

# Grid & Breakpoints

Breakpoints define responsive layout boundaries. They are exported as **TypeScript constants** (not CSS custom properties) because CSS `var()` cannot be used inside `@media` queries.

> **Note:** Breakpoints are primitive-only tokens. The theme switcher has no effect on this page.

## Breakpoint Scale

<BreakpointRuler breakpoints={BREAKPOINTS} />

## TypeScript Usage

Import breakpoint constants in your application code:

```tsx
import { GridBreakpointSm, GridBreakpointMd, GridBreakpointLg } from '@design-system-x/tokens'

const query = `@media (min-width: ${GridBreakpointMd})`
```

## Token Reference

| Name | TS Constant | CSS Token | Value |
|------|------------|-----------|-------|
| sm | `GridBreakpointSm` | N/A (not a CSS var) | 640px |
| md | `GridBreakpointMd` | N/A (not a CSS var) | 768px |
| lg | `GridBreakpointLg` | N/A (not a CSS var) | 1024px |
| xl | `GridBreakpointXl` | N/A (not a CSS var) | 1280px |
| 2xl | `GridBreakpoint2xl` | N/A (not a CSS var) | 1536px |
```
