This task requires human interaction with GitHub and Figma UIs that Claude cannot access. Plan 01 authored all semantic token JSON files and $themes.json in the repository. These need to be pushed to GitHub so Tokens Studio Pro can access them, then pushed from Tokens Studio Pro into Figma to create the Semantic Variable Collection.

**Steps for the human to complete:**

1. **Create GitHub repository** (if not already done):
   - Go to github.com/new
   - Name: design-system-x (or your preferred name)
   - Visibility: private (recommended)
   - Do NOT initialize with README (repo already has content)

2. **Push to GitHub:**
   ```bash
   cd "/Users/merlinzuni/develop/design system tokens"
   git remote add origin https://github.com/merlinzuni/<repo-name>.git
   git push -u origin main
   ```

3. **Create GitHub Fine-Grained PAT:**
   - GitHub Settings -> Developer settings -> Personal access tokens -> Fine-grained tokens
   - Token name: "Tokens Studio Pro"
   - Repository access: Only select repositories -> select the design system repo
   - Permissions: Contents -> Read and write
   - Copy the token

4. **Configure Tokens Studio Pro in Figma:**
   - Open the Figma file for the design system
   - Open Tokens Studio Pro plugin
   - Go to Settings -> Sync providers -> Add new -> GitHub
   - Enter: PAT (from step 3), Repository owner: merlinzuni, Repository name: the repo name, Branch: main, File path: `packages/tokens/tokens/`
   - Storage: choose "Folder" (NOT "Single file" — required for Pro themes feature per Research Pitfall 5)
   - Click "Pull from GitHub" to load the token JSON files into the plugin

5. **Push to Figma (bootstrap step per D-15):**
   - In Tokens Studio Pro, go to Themes view
   - Verify 4 themes are detected from $themes.json: parent-brand/light, parent-brand/dark, child-brand/light, child-brand/dark
   - Use "Export to Figma" with the themes-based export (NOT token-sets export)
   - This creates the `Semantic` Variable Collection in Figma with 4 modes automatically

6. **Verify in Figma:**
   - Open Figma Variables panel (right sidebar or Variables button)
   - Confirm `Semantic` collection exists
   - Confirm it has exactly 4 modes: parent-brand/light, parent-brand/dark, child-brand/light, child-brand/dark
   - Confirm variable names use slash notation (e.g., `color/background/default`, `color/text/primary`)
   - Spot-check: `color/action/default` in parent-brand/light mode should reference the teal brand.500 primitive

7. **Verify round-trip sync:**
   - In Figma, change any one semantic variable value (e.g., change `color/border/subtle` to a different primitive alias)
   - In Tokens Studio Pro, "Push to GitHub"
   - Check the GitHub repo — a new commit should appear with the changed JSON
   - Revert the change in Figma and push again to restore original state
  </action>
